# Rebuilding the Wasm runtime and toolchain

This repo does **not** commit `dist/edgejs.{js,wasm}` or `build/edge`. Everyone‚Äî**CI, a Linux VM (e.g. Cory on EC2), or a Windows dev machine**‚Äîrecreates artifacts the same way: **Emscripten + Make + pinned patch**.

**Do not stop at the build** on Cory: the target is **CI parity**‚Äîinstall Chromium, set **`CHROME_BIN`**, run **`npm ci`**, then **`npm run test:nodejs-in-tab-contract`** and optionally **`make test-integration`** (see [Run tests on Cory / EC2](#run-tests-on-cory--ec2-parity-with-ci)).

## What ‚Äúentire toolchain‚Äù means here

| Piece | Version / source | Role |
|--------|------------------|------|
| **Node.js** | ‚â• 18 (CI uses **22**) | Tests, `napi-bridge`, dev server |
| **Git + CMake + Python 3** | OS or package manager | Configuring EdgeJS (CMake) |
| **Emscripten** | **3.1.64** (match CI) | `emcmake` / `emcc` cross-build |
| **EdgeJS source** | `https://github.com/wasmerio/edgejs.git` (`edgejs-src/`) | Upstream runtime |
| **NAPI headers** | Submodule `edgejs-src/napi` | `EDGE_NAPI_PROVIDER=imports` (HTTPS URL forced in `Makefile` **fetch**) |
| **Emscripten patch** | `patches/edgejs-emscripten.patch` | OpenSSL/libuv/CLI CMake fixes for `Emscripten` |

**Outputs you must have** for integration tests and the in-tab contract:

- `dist/edgejs.wasm`, `dist/edgejs.js`
- `build/edge` (CommonJS loader stub generated by `make build`)

## Canonical local command sequence

From the **repository root**, with Emscripten **activated** in the shell:

```bash
make fetch      # clone / update EdgeJS; init napi submodule over HTTPS
source "${EMSDK}/emsdk_env.sh"   # Linux/macOS: adjust path to your emsdk install
make configure  # clean edgejs-src, apply patch, CMake configure ‚Üí build/
make build      # compile; populate dist/ + build/edge
```

Verify:

```bash
test -f dist/edgejs.wasm && test -f dist/edgejs.js && test -f build/edge && echo OK
```

After artifacts exist, run **tests** (next section on EC2, or `npm ci && npm run test:nodejs-in-tab-contract` locally).

## Run tests on Cory / EC2 (parity with CI)

CI‚Äôs **Integration** job does two things in order: **build Wasm** (`make configure` / `make build`), then **run tests** (`make test-integration`, which includes `npm run test:nodejs-in-tab-contract` and `make test-soak-quick`). On Cory‚Äôs machine, mirror that.

### Packages (Debian/Ubuntu-style)

- **Node.js** **22** if you can (matches `ci.yml`): e.g. `nvm install 22` or your distro.
- **Chromium** for Phase 1 of the unified contract (Playwright launches this binary):  
  `sudo apt-get update && sudo apt-get install -y chromium-browser || sudo apt-get install -y chromium`
- **Build / configure**: `build-essential`, `cmake`, `python3`, `git`, `make`, (`ninja-build` optional).

### Emscripten

Install and activate **emsdk 3.1.64** ([Emscripten downloads](https://emscripten.org/docs/getting_started/downloads.html)). Every **configure/build** terminal:

```bash
source "$EMSDK/emsdk_env.sh"
```

### Repo + npm

```bash
git clone <repo-url> && cd <repo>
git checkout dev    # or the branch under test
npm ci
```

### Tell Playwright which Chromium to use

Phase 1 reads **`CHROME_BIN`** (see `tests/helpers/contract-phase-browser.mjs`). Set it in the same shell as tests:

```bash
export CHROME_BIN="$(command -v chromium-browser || command -v chromium || command -v google-chrome || true)"
test -x "$CHROME_BIN" || { echo "Install Chromium and set CHROME_BIN"; exit 1; }
```

### Wasm artifacts ‚Äî build **or** restore from CI

**Option A ‚Äî build on Cory (full toolchain)**


### CI (GitHub Actions)

- **Integration job** (`.github/workflows/ci.yml`): after the quick tier, runs `make configure` + `make build` with emsdk **3.1.64**, uploads **`edgejs-wasm-<sha>`** (`dist/edgejs.wasm`, `dist/edgejs.js`, `build/edge`), then runs **`make test-integration`** and **`make test-soak-quick`** (same sequence Cory should mirror after producing artifacts).
- **Manual Wasm-only rebuild** (`.github/workflows/wasm-rebuild.yml`): **Actions ‚Üí ‚ÄúWasm runtime rebuild‚Äù ‚Üí Run workflow** ‚Äî builds and uploads the same artifact **without** requiring the quick tier (for emergency recovery).
- **Pages** (`.github/workflows/pages.yml`): same `make configure` / `make build` before assembling `docs/`.

**Restoring binaries from CI:** open a successful run ‚Üí **Artifacts** ‚Üí download `edgejs-wasm-‚Ä¶`, unzip into the repo root so paths match (`dist/‚Ä¶`, `build/edge`).

## npm scripts

- `npm run build` runs `build-emscripten.sh`, which **delegates to** `make fetch configure build` (same as above).
- `npm run build:debug` uses `BUILD_TYPE=Debug`.

Legacy layouts that used `src/` and root-level `edgejs.wasm` are obsolete; the **Makefile** layout is authoritative.

## If upstream EdgeJS moves

If `make configure` fails because `patches/edgejs-emscripten.patch` no longer applies:

1. Reset `edgejs-src` to a known `origin/main` commit.
2. Re-generate the patch from a clean tree (apply edits, then e.g. `git -C edgejs-src diff > patches/edgejs-emscripten.patch`).
3. Land the updated patch in this repo; bump CI emsdk only if required by upstream.

Optional reproducibility: after `make fetch`, `git -C edgejs-src checkout <sha>` to pin EdgeJS before `make configure`.

## Troubleshooting

| Symptom | Likely fix |
|--------|------------|
| **`wasm-ld` SIGSEGV** in `lld::wasm::ImportSection::addImport` at the **final link** (Emscripten **3.1.64** / **LLD 19**) | Confirmed on some hosts (e.g. **Ubuntu 24.04**, **kernel 6.17**, official `wasm-binaries` tarball). Not a corrupt download (same `md5` after re-install), not ASLR (`setarch -R` still crashes). **Workaround:** use a **newer SDK** so you get **LLD ‚â• 22**, e.g. `cd ~/emsdk && ./emsdk install tot && ./emsdk activate tot`, then clean and `make configure && make build`. CI still pins **3.1.64**; if your machine needs `tot`, use that locally until LLVM 19 catches up. |
| **`missing build/edge.js`** or tiny broken **`build/edge`** | Fixed in-repo: Emscripten often emits **`-o edge`** as a large **`build/edge`** JS file; the Makefile now renames it to **`edge.js`** and writes the Node shim via **`scripts/write-build-edge-shim.mjs`**. |
| Contract **Phase 1** fails / browser won‚Äôt start | Set **`CHROME_BIN`** to Chromium; install `chromium` / `chromium-browser`; run `npm ci` so `playwright-core` is present. |
| `Vendored OpenSSL is not yet wired for native target system 'Emscripten'` | Patch not applied; ensure `make configure` succeeds (no ‚Äúpatch failed‚Äù message). |
| Missing `dist/edgejs.wasm` | Run `make build`; confirm `build/edge.wasm` exists and copy step in `Makefile` ran. |
| Submodule / `napi` missing | Run `make fetch` again. |
| `test-soak-quick` / CI soak step | Defined in `Makefile` as `npm run test:soak:quick`. |

# Plan 02-01: N-API Bridge Hardening (Bootstrap + Runtime Correctness)

<objective>
Harden the N-API bridge so the Wasm runtime can bootstrap and execute user code with correct error propagation and stable handle lifecycle behavior.

This plan focuses on Phase 2 runtime correctness first: import coverage, error semantics, callback/property behavior, and deterministic execution probes (`eval` and `runFile`).
</objective>

<work_completed>
## Work Completed So Far

1. **Import Coverage + Unofficial APIs**
   - Added missing N-API and unofficial hooks needed during bootstrap (contextify, module_wrap callbacks, promise/microtask helpers, property/value helpers, error creators/throwers).

2. **Error + Handle Semantics**
   - Strengthened last-error + pending-exception paths.
   - Corrected reference/handle behavior for wrapped native pointers using object identity, reducing lifecycle mismatches.

3. **Runtime Probe Reliability**
   - Stabilized `initEdgeJS` with deterministic startup options.
   - Made `eval` and `runFile` probe paths deterministic via wrapper scripts in MEMFS, so probes return reliably with explicit status.
</work_completed>

# Phase 3 Plan 03-01: Core Module Foundation

**Phase:** 3 — Core Modules + Filesystem
**Plan:** 1 of 2
**Focus:** EventEmitter, Buffer, Streams, process, util — the module primitives everything else depends on
**Requirements:** CORE-01, CORE-02, CORE-03, CORE-04, CORE-05, CORE-06, CORE-07, CORE-08

## Objective

Deliver production-grade implementations of the 8 core Node.js module contracts that Claude Code depends on. Every module in this plan is a dependency of Phase 4 (networking needs streams), Phase 5 (subprocess needs stdio pipes), and Phase 6 (terminal needs process.stdin/stdout).

## Implementation Order (Dependency-Driven)

### Step 1: Module Registration Hook
**What:** Wire `_registerBuiltinOverride` into EdgeJS module resolution so browser-side modules actually replace Wasm-compiled ones.
**Why:** Everything in `browser-builtins.js` is dead code until this hook works. This is the single blocker for all browser-first overrides.
**Files:** `napi-bridge/index.js` (module resolution path), `napi-bridge/browser-builtins.js` (registration call)
**Exit:** `registerBrowserBuiltins(instance)` succeeds and `require('path')` returns `pathBridge`.

### Step 2: EventEmitter (CORE-01)
**What:** Browser-native EventEmitter implementing `on`, `off`, `once`, `emit`, `removeListener`, `removeAllListeners`, `listenerCount`, `prependListener`.
**Why:** Used by every stream, every I/O object, process itself. Must exist before streams.
**Files:** New: `napi-bridge/eventemitter.js`; Update: `browser-builtins.js` (register as `events` module)
**Exit:** EventEmitter passes conformance: on/emit, once fires once, removeListener works, error event throws if unhandled.

### Step 3: Buffer Enhancement (CORE-02)
**What:** Extend `bufferBridge` with `toString(encoding)`, `slice()`, `copy()`, `compare()`, `write()`, `byteLength()`, `indexOf()`. Full utf8/base64/hex/ascii/latin1 encoding support.
**Why:** Claude Code reads file contents as Buffers constantly. `toString()` is called on every file read.
**Files:** `napi-bridge/browser-builtins.js` (extend `bufferBridge`)
**Exit:** `Buffer.from('hello').toString('hex')` roundtrips. All encoding pairs work.

### Step 4: Streams (CORE-03)
**What:** Readable, Writable, Transform, Duplex with backpressure and `pipe()`.
**Why:** fs.createReadStream, process.stdin/stdout, HTTP response bodies, subprocess stdio — all are streams.
**Files:** New: `napi-bridge/streams.js`; Update: `browser-builtins.js` (register as `stream` module)
**Depends on:** EventEmitter (Step 2), Buffer (Step 3)
**Exit:** `readable.pipe(writable)` transfers data with backpressure. Transform processes chunks. Duplex works bidirectionally.

### Step 5: process Enhancement (CORE-04, CORE-05)
**What:**
- `process.env` injectable via `initEdgeJS()` options
- `process.cwd()` returns configurable virtual working directory
- `process.stdin` as a Readable stream
- `process.stdout` / `process.stderr` as Writable streams (connected to terminal in Phase 6)
- `process.on('exit', fn)` lifecycle events
**Files:** `napi-bridge/browser-builtins.js` (extend `processBridge`), `napi-bridge/index.js` (`initEdgeJS` options)
**Depends on:** Streams (Step 4), EventEmitter (Step 2)
**Exit:** `process.stdin.on('data', cb)` receives input. `process.stdout.write('hello')` outputs. `process.env.HOME` returns injected value.

### Step 6: path/url/util Polish (CORE-06, CORE-07, CORE-08)
**What:**
- `path`: Already complete in `pathBridge` — add `path.posix` alias, `path.win32` stub
- `url`: Already complete — add `url.fileURLToPath`, `url.pathToFileURL`
- `util`: `promisify`, `inherits`, `inspect` (basic), `types.isPromise/isDate/isRegExp`
**Files:** `napi-bridge/browser-builtins.js` (extend existing bridges), New: `napi-bridge/util.js`
**Exit:** `util.promisify(fs.readFile)` works. `url.fileURLToPath('file:///tmp/f')` returns `/tmp/f`.

## Risks

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Module resolution hook harder than expected | HIGH | May need to intercept at N-API `napi_get_named_property` level, not just JS |
| Streams backpressure complexity | MEDIUM | Start with simple flowing mode; add paused mode incrementally |
| Buffer encoding edge cases | LOW | Use TextEncoder/TextDecoder for utf8; manual for hex/base64 (already partially done) |
| process.stdin without terminal | LOW | Phase 3 wires the stream; Phase 6 connects it to xterm.js input |

# Phase 3 Plan 03-02: Filesystem (MEMFS)

**Phase:** 3 — Core Modules + Filesystem
**Plan:** 2 of 2
**Focus:** In-memory filesystem with full POSIX-like API surface
**Requirements:** FS-01, FS-02, FS-03, FS-04, FS-05, FS-06, FS-07
**Depends on:** Plan 03-01 (Streams for createReadStream/createWriteStream, Buffer for file content, EventEmitter for fs.watch)

## Objective

Deliver a complete `fs` module backed by MEMFS that supports both sync and async variants of all 7 required filesystem operations. Claude Code reads files, writes files, lists directories, and checks file stats on every single turn. This is not optional infrastructure — it is the primary surface Claude Code operates on.

## Architecture

```
┌──────────────────────────────────┐
│         fs API (browser)         │  ← Node.js fs contract
├──────────────────────────────────┤
│       MEMFS (in-memory tree)     │  ← inode-based file tree in JS
├──────────────────────────────────┤
│    Browser Builtin Override      │  ← registered via _registerBuiltinOverride
└──────────────────────────────────┘
```

**MEMFS design:**
- Tree of inodes: each inode is `{ type: 'file'|'dir', content: Uint8Array|null, children: Map, stat: {...} }`
- Paths resolved via POSIX rules (no Windows paths in browser)
- `stat` tracks: size, mtime, ctime, atime, mode, isFile(), isDirectory()
- File descriptors for open/read/write/close (needed by some Claude Code paths)

## Implementation Steps

### Step 1: MEMFS Core
**What:** In-memory inode tree with resolve, lookup, create, delete primitives.
**Files:** New: `napi-bridge/memfs.js`
**Exit:** Internal API: `memfs.resolve('/a/b/c')`, `memfs.createFile('/f', data)`, `memfs.readFile('/f')`, `memfs.mkdir('/d')`, `memfs.stat('/f')`.

### Step 2: fs.readFile / fs.readFileSync (FS-01)
**What:** Read file content as Buffer or string (with encoding option).
**Exit:** `fs.readFileSync('/hello.txt', 'utf8')` returns string. Async variant calls callback.

### Step 3: fs.writeFile / fs.writeFileSync (FS-02)
**What:** Create or overwrite files. Support string and Buffer data, encoding option.
**Exit:** `fs.writeFileSync('/f', 'data')` then `fs.readFileSync('/f', 'utf8')` returns `'data'`.

### Step 4: fs.readdir / fs.readdirSync (FS-03)
**What:** List directory contents. Support `withFileTypes` option returning Dirent objects.
**Exit:** After creating files, `fs.readdirSync('/')` returns array of names.

### Step 5: fs.stat / fs.statSync (FS-04)
**What:** Return Stats object with size, mtime, `isFile()`, `isDirectory()`.
**Exit:** `fs.statSync('/f').isFile()` returns `true`. `size` matches content length.

### Step 6: fs.mkdir / fs.mkdirSync (FS-05)
**What:** Create directories. Support `{ recursive: true }` option.
**Exit:** `fs.mkdirSync('/a/b/c', { recursive: true })` creates all intermediate dirs.

### Step 7: fs.unlink / fs.unlinkSync (FS-06)
**What:** Delete files. Throw ENOENT for missing files, EISDIR for directories.
**Exit:** After unlink, stat throws ENOENT.

### Step 8: fs.rename / fs.renameSync (FS-07)
**What:** Move/rename files and directories.
**Exit:** After rename, old path throws ENOENT, new path returns content.

### Step 9: Additional fs APIs (Claude Code needs)
**What:** These aren't in the requirements but Claude Code uses them:
- `fs.existsSync(path)` — used constantly for file existence checks
- `fs.createReadStream(path)` / `fs.createWriteStream(path)` — streaming file I/O
- `fs.promises.*` — promise-based variants of all operations
- `fs.accessSync(path)` — permission checks
- `fs.realpathSync(path)` — symlink resolution (identity in MEMFS)
- `fs.openSync` / `fs.readSync` / `fs.writeSync` / `fs.closeSync` — fd-based I/O
**Depends on:** Streams (Plan 03-01 Step 4)

### Step 10: Pre-populate Working Directory
**What:** `initEdgeJS()` accepts a `files` option to pre-populate MEMFS with project files.
**Files:** `napi-bridge/index.js` (initEdgeJS options), `napi-bridge/memfs.js`
**Exit:** `initEdgeJS({ files: { '/project/index.js': 'console.log("hi")' } })` makes file available via fs.readFileSync.

## Risks

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Memory pressure from large files | HIGH | Streaming API (createReadStream) processes chunks; warn at 100MB total MEMFS |
| fd table management complexity | MEDIUM | Simple integer counter; Map<fd, { inode, position, flags }> |
| fs.promises API surface | LOW | Thin wrapper: `return new Promise((resolve, reject) => fs.readFile(p, cb))` |
| Path edge cases (trailing slashes, .., symlinks) | LOW | POSIX normalize all paths at entry; no symlinks in v1 |

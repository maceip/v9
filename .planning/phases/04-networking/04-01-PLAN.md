# Phase 4 Plan 04-01: Networking (fetch proxy + SSE + crypto)

**Phase:** 4 — Networking
**Plan:** 1 of 1
**Focus:** HTTPS via browser fetch, streaming SSE responses, crypto completion
**Requirements:** NET-01, NET-02, NET-03, NET-04, NET-05
**Depends on:** Phase 3 (Streams, Buffer, EventEmitter)

## Objective

Claude Code can make Anthropic API calls and receive streaming responses in the browser. This is the single requirement that turns the runtime from a demo into a functional tool.

## Implementation Steps

### Step 1: Determine SDK Transport

**Before writing code, answer this question:**
Does the Anthropic SDK (`@anthropic-ai/sdk`) use `fetch()` or `http.request()`?

```js
// Check the SDK source:
// node_modules/@anthropic-ai/sdk/src/core.ts or similar
// Look for: import http from 'http' vs fetch()
```

- If the SDK uses `fetch()` → browser already has it natively. Skip the http/https override entirely. Go to Step 3.
- If the SDK uses `http.request()` → must implement http/https module override (Step 2).
- If the SDK supports both → configure it to use `fetch()`.

**This decision changes the scope of Phase 4 dramatically.** Do not skip it.

### Step 2: http/https Module Override (only if needed)

**What:** Replace Node.js `http`/`https` modules with fetch-based implementations.
**Files:** New: `napi-bridge/http.js`

The override must implement:
```
http.request(url, options, callback) → ClientRequest
http.get(url, options, callback) → ClientRequest
https.request(url, options, callback) → ClientRequest
https.get(url, options, callback) → ClientRequest
```

**ClientRequest** (extends Writable from Phase 3):
```
req.write(chunk)      → accumulate body
req.end(chunk)        → fire fetch() with accumulated body
req.setHeader(k, v)   → store header
req.getHeader(k)      → retrieve header
req.on('response', cb) → cb receives IncomingMessage
req.on('error', cb)    → cb receives Error
req.abort()           → AbortController.abort()
```

**IncomingMessage** (extends Readable from Phase 3):
```
res.statusCode        → HTTP status (200, 404, etc.)
res.statusMessage     → HTTP status text
res.headers           → lowercased header object
res.on('data', cb)    → body chunks as Buffers
res.on('end', cb)     → body complete
```

**How it works internally:**
```
req.end() →
  1. Build Request from accumulated headers + body
  2. Call fetch(url, { method, headers, body })
  3. Wrap response in IncomingMessage
  4. If response.body is ReadableStream → pipe chunks to IncomingMessage
  5. Emit 'response' event with IncomingMessage
```

### Step 3: Streaming SSE Bridge (NET-02)

**What:** Browser `ReadableStream` → Node.js `Readable` bridge for streaming API responses.
**Why:** Anthropic API returns `text/event-stream` — each chunk is an SSE event. Claude Code reads these as a Node.js Readable stream.
**Files:** New: `napi-bridge/fetch-streams.js`

```js
/**
 * Bridge a browser ReadableStream to a Node.js Readable.
 * Used by both the http override (if needed) and direct fetch() calls.
 */
function webStreamToNodeReadable(webStream) {
  const reader = webStream.getReader();
  return new Readable({
    async read() {
      const { done, value } = await reader.read();
      if (done) {
        this.push(null);
      } else {
        this.push(Buffer.from(value));
      }
    },
  });
}
```

This is the critical bridge. If this works, streaming works. If it doesn't, Claude Code hangs waiting for response data.

**SSE specifics:**
- Content-Type: `text/event-stream`
- Events separated by `\n\n`
- Each event has `event:`, `data:`, `id:` fields
- The Anthropic SDK handles SSE parsing internally — we just need to deliver the raw bytes as a Readable stream

### Step 4: Crypto Completion (NET-03, NET-04)

**What:** Make `createHash` synchronous and add `createHmac`.
**Files:** `napi-bridge/browser-builtins.js` (extend `cryptoBridge`)

Current `createHash` is async (uses `crypto.subtle.digest`). Node.js `createHash` is sync. Options:
- **Option A:** Pre-compute with sync fallback. Use `crypto.subtle` in a synchronous wrapper (not possible — Web Crypto is async-only).
- **Option B:** Pure JS SHA implementation. Use a bundled SHA-256/384/512 in pure JS. ~200 lines, runs sync.
- **Option C:** If the SDK always `await`s the hash, async is fine. Check SDK usage.

**Recommendation:** Option C first (check if async works). If not, Option B.

**createHmac** (NET-04):
```js
createHmac(algorithm, key) {

```

### Step 5: Header Pass-Through (NET-05)

**What:** Ensure HTTP request/response headers are preserved correctly.
**Key headers for Anthropic API:**
```
Request:
  Authorization: Bearer sk-ant-...
  Content-Type: application/json
  Accept: text/event-stream (for streaming)
  anthropic-version: 2023-06-01
  x-api-key: sk-ant-... (alternative auth)

Response:
  content-type: text/event-stream
  x-request-id: ...
  (all lowercased in Node.js convention)
```

**CORS:** Browser fetch enforces CORS. Anthropic API must return `Access-Control-Allow-Origin`. If it doesn't, the browser will block the request. This may require a CORS proxy. **This is a potential blocker — investigate early.**

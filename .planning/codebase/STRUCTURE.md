## Directory Purposes

**napi-bridge/:**
- Purpose: Implement N-API spec for browser JS engine; bridge Wasm ↔ browser
- Contains: JavaScript implementation of N-API host functions
- Key files:
  - `index.js`: Core NapiBridge class (450 lines)
  - `browser-builtins.js`: Overrideable module implementations (300 lines)
  - `jspi-adapter.js`: Async I/O via WebAssembly.Suspending (192 lines)

**wasi-shims/:**
- Purpose: Fill libc/V8 API gaps for EdgeJS compiled to Wasm+Emscripten
- Contains: C++ header files with type stubs, namespace fixes, function declarations
- Key files:
  - `wasi-shims-index.h`: Master include; lists all shims
  - `wasi-v8-missing-types.h`: V8 type definitions (Isolate, Context, etc.)
  - `wasi-v8-internals.h`: V8 internal structures required by runtime
  - `wasi-filesystem-stubs.h`: Minimal POSIX file descriptor layer
  - ~20 additional specialized shims for V8 operators, bit operations, etc.

**edgejs-src/:**
- Purpose: Node.js runtime compiled to Wasm; provides ~46 built-in modules
- Contains: Full Node.js module implementations in C++
- Key directories:
  - `src/`: Core runtime (process, event loop, module loader)
  - `lib/`: JavaScript implementations of Node modules (readable, writable, etc.)
  - `napi/`: N-API specification headers + test harness
  - `deps/`: Vendored C libraries (libuv, OpenSSL, nghttp2, icu, zlib, v8)

**tests/:**
- Purpose: Validate N-API bridge functionality and module integration
- Contains: Node.js-runnable test harnesses
- Covers: Handle table lifecycle, type marshaling, import generation

**dist/:**
- Purpose: Build output directory
- Contains: Final Wasm binary, Emscripten JS wrapper, worker script
- Generated by: `./build-emscripten.sh` or `make build`

## Key File Locations

**Entry Points:**

- `package.json`: Main export = `napi-bridge/index.js`
- `napi-bridge/index.js`: Exports `initEdgeJS(options)` async function
  - Call this to initialize EdgeJS runtime in browser
  - Returns `{ eval(code), runFile(path), fs, module }`

- `dist/edgejs.js`: Emscripten module wrapper
  - Generated by Emscripten; acts as intermediary for Wasm instantiation
  - Exports async function that returns Wasm instance + runtime

**Configuration:**

- `emscripten-toolchain.cmake`: Toolchain for CMake cross-compilation to Wasm
  - Sets compiler to `emcc`, linker flags for JSPI/threading/memory
  - Exports functions: `_edge_napi_init`, `_malloc`, `_free`

- `build-emscripten.sh`: Build automation wrapper
  - Calls `emcmake cmake` to configure
  - Calls `cmake --build` to compile

- `Makefile`: High-level targets (setup, fetch, configure, build, test)
  - Variables: `EMSDK_DIR`, `BUILD_TYPE`, `JOBS`
  - Downloads/installs Emscripten if missing

**Core Logic:**

- `napi-bridge/index.js:NapiBridge`: Handle table + N-API implementations
  - `createHandle()`: Allocate integer ↔ JS value mapping
  - `getImports()`: Return import object for WebAssembly.instantiate()
  - `readString()` / `writeString()`: UTF-8 marshaling

- `napi-bridge/browser-builtins.js`: Module overrides
  - `cryptoBridge`: Web Crypto API wrapper
  - `pathBridge`: Path utilities
  - `urlBridge`: URL parsing
  - `bufferBridge`: Uint8Array with Node.js Buffer API

- `napi-bridge/jspi-adapter.js`: Async I/O imports
  - `__wasi_sock_connect()`: TCP via WebTransport
  - `__wasi_sock_recv()` / `__wasi_sock_send()`: Network I/O
  - `__wasi_poll_oneoff()`: Event loop via setTimeout

- `edgejs-src/src/edge_runtime.cc`: Wasm runtime bootstrap
  - Initializes libuv event loop
  - Registers ~46 Node.js built-in modules
  - Implements `edge_eval()` and `edge_run_file()` exports

**Testing:**

- `tests/test-basic.mjs`: NapiBridge unit tests
  - Handle table creation/freeing
  - Handle scope lifecycle
  - Well-known handle behavior (0=undefined, 1=null, 2=global)

- `tests/test-napi-bridge.mjs`: Extended N-API tests
  - Import object generation
  - Value creation functions (int32, string, object, array)
  - Type checking, property access, function calls

## Naming Conventions

**Files:**

- `*-toolchain.cmake`: Cross-compile configuration
- `build-*.sh`: Build scripts (e.g., `build-emscripten.sh`)
- `edge_*.cc` / `edge_*.h`: EdgeJS C++ modules
- `wasi-*.h`: WASI shim headers
- `test-*.mjs`: Test harnesses (Node.js ESM)
- `.js` for ESM, `.mjs` for explicit ESM (no ambiguity)

**Directories:**

- `*-bridge/`: Adapter/translation layer
- `wasi-*`: POSIX/WASI compliance layer
- `edgejs-src/`: Upstream source (convention: `-src` suffix)
- `dist/`: Build output (conventional name)

**Functions:**

- N-API imports: `napi_*` (e.g., `napi_create_string_utf8`)
- WASI syscalls: `__wasi_*` (e.g., `__wasi_sock_connect`)
- EdgeJS exports: `edge_*` (e.g., `edge_eval`)
- Class methods: camelCase (e.g., `createHandle`, `getHandle`)
- Classes: PascalCase (e.g., `NapiBridge`)

**Variables:**

- Handle/pointer variables: `*Handle`, `*Ptr` suffix
- Bridge/runtime references: `bridge`, `wasm`, `instance`
- Module/instance: `module`, `instance`

## Where to Add New Code

**New Builtin Module Override (crypto, path, etc.):**
- Primary code: `napi-bridge/browser-builtins.js`
  - Add export object with module name key (e.g., `cryptoBridge`)
  - Implement methods matching Node.js API
- Registration: `registerBrowserBuiltins()` function at end of file
  - Maps module name to exported object
- Example: Replacing Node's `fs` module
  - Would add `fsBridge` object with `readFile()`, `writeFile()`, etc.
  - Only practical for read-only or virtual FS operations

**New JSPI Import (new syscall type):**
- Primary code: `napi-bridge/jspi-adapter.js`, `createIOImports()` function
- Wrap with `wrapAsyncImport(asyncFn)` to enable JSPI suspension
- Return appropriate WASI errno on failure
- Example: WebSocket support
  - Add `__wasi_sock_connect_ws()` that uses `new WebSocket()`
  - Handle `onopen`, `onerror`, `onmessage`

**New WASI Shim Header:**
- Location: `wasi-shims/wasi-*.h`
- Pattern: Include guard, then type/function declarations
- Master include: Add `#include` line to `wasi-shims-index.h`
- Example: If EdgeJS needs a missing C++ operator
  - Create `wasi-v8-custom-operator.h`
  - Define `operator[]()`, `operator<()`, etc. as inline stubs
  - Include from `wasi-shims-index.h`

**New Test Case:**
- Unit tests: `tests/test-basic.mjs`
  - Add `test()` call with name and assertion function
  - Access bridge instance via `const bridge = new NapiBridge(null)`
  - No Wasm required; tests JavaScript-only functionality

- N-API tests: `tests/test-napi-bridge.mjs`
  - Set up memory buffer: `const memoryBuffer = new ArrayBuffer(4096)`
  - Call imports directly: `imports.napi_create_int32(env, value, ptr)`
  - Validate with `bridge.getHandle()` and assertions

**New Build Target (Makefile):**
- Add `.PHONY` entry for new target
- Add target rule with dependencies
- Example: `make doc` target
  - Dependency: source files
  - Command: `doxygen Doxyfile` or similar
  - Targets follow convention: `all`, `setup`, `clean`, `distclean`

## Special Directories

**dist/:**
- Purpose: Build output directory
- Generated: Yes (by `build-emscripten.sh` or `cmake --build`)
- Committed: No (should be in `.gitignore`)
- Contains:
  - `edgejs.wasm`: Compiled binary (~30-50 MB uncompressed, ~10 MB gzipped)
  - `edgejs.js`: Emscripten wrapper (~50-100 KB)
  - `edgejs.worker.js`: Worker entrypoint (if using Workers)

**build/ (or build-emscripten/):**
- Purpose: CMake build directory (intermediate files)
- Generated: Yes
- Committed: No
- Contains: Object files, CMake cache, build logs

**edgejs-src/ (submodule):**
- Purpose: Upstream EdgeJS source
- Generated: No (cloned via `git submodule` or manual clone)
- Committed: Yes (as submodule reference)
- Update: `git submodule update --init --recursive`

**.planning/codebase/:**
- Purpose: GSD codebase documentation
- Generated: By `/gsd:map-codebase` command
- Committed: Yes
- Contains: ARCHITECTURE.md, STRUCTURE.md, etc.
